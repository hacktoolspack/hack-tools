1- Quoi de neuf dans RemoteExec 5.03 (Septembre 2012)
-----------------------------------------------------------------
- Correction : Lorsque le répertoire temporaire sur les machines cibles était défini (avec la variable d'environnement %TEMP%) dans une autre partition que C:\, RemoteExec copiait les fichiers exécutables dans le mauvais répertoire générant une erreur 0x3.
- Correction : Lorsque le répertoire destination d'une opération de fichier se trouvait sur une partition différente de c:\, RemoteExec utilisait un chemin UNC incorrect pour transférer/supprimer le fichier/répertoire.
- Correction : Lors d'une tentative de chargement d'une liste de machines depuis un fichier en cours d'utilisation dans un autre processus, RemoteExec crashait.
- Correction : Si l'AD contenait une OU avec un nom distingué de plus de 512 charactères RemoteExec crashait lors du démarrage.
- Correction : RemoteExec crashait au démarrage sous Windows 8

2- Quoi de neuf dans RemoteExec 5.02 (Novembre 2011)
----------------------------------------------------
- Correction : Lorsqu'une clé de licence invalide était saisie un mauvais message était affiché.
- Correction : Si des guillemets (") étaient insérés dans le message de popup ou de redémarrage le message était tronqué.
- Correction : Lors de l'installation de package msi les arguments n'étaient pas pris en compte
- Correction : L'execution à distance d'un batch avec des parenthèses dans le nom échouait car le chemin du batch n'était pas mis entre guillemets par RemoteExec.
- Ajout : Support du déploiement du framework .net 4 avec l'action "Installation de mise a jour"
- Correction : Pour l'action Installation de mise a jour, si uniquement si nécessaire était sélectionné, tout filtre additionnel configuré à la main était ignoré.
- Ajout : Possibilité d'utiliser l'authentification SMTP et le cryptage SSL pour l'envoit des rapports par E-mail.
- Correction : Si les arguments contenaient la séquence \" (anti-slash suivit d'un guillemet) l'exécution pouvait échouer.

3- Quoi de neuf dans RemoteExec 5.01 (Février 2011)
---------------------------------------------------
- Correction : RemoteExec crashait sur les machines Windows 2000
- Correction : L'import de fichiers action (*.rea) d'anciennes versions de RemoteExec crashait
- Correction : Si le même ordinateur était rajouté plusieurs fois d'une façon différente dans la liste des ordinateurs cibles une exception pouvait se produire.
- Correction : "Sauvegarder dans mes ..." n'enregistrait pas une version à jour de la configuration dans les favoris.
- Correction : Lorsqu'une tâche planifiée était renommé le chemin vers le fichier XML de configuration n'était pas modifié avec pour conséquence quel le Job à distance n'était plus exécuté aux heures planifiées.
- Correction : Lorsqu'une tâche planifiée était renommé successivement deux fois une exception était générée.
- Correction : Des exécutions distantes simultanées sur une même machine pouvait échouer.
- Correction : Problème de base de données en cas d’exécutions distantes simultanées.
- Correction : Une action multiple démarrant par un réveil générait des exceptions de base de données
- Correction : Un chemin d'exécutable UNC de plus de 128 caractères faisait crasher RemoteExec
- Correction : Le filtre de langue ne fonctionnait pas
- Correction : La configuration d'une action multiple sauvegardée dans un dossier favori ou dans une tâche planifiée ne gardait pas les modifications dans certains cas.
- Correction : Une action popup ne pouvait pas être enregistrée dans "Mes Actions à Distance"

4- Quoi de neuf dans RemoteExec 5 version finale (Décembre 2010)
----------------------------------------------------------------
- Correction : Les adresses MAC scannées avec l'action système "Rapatrie les infos de réveil" s'affichaient dans la liste des adresses ajoutées manuellement au lieu de la liste des adresses MAC ajouté automatiquement.
- Correction : Lorsque le menu contextuel était invoqué sur un nœud de l'arborescence de configuration qui n'avait pas le focus, l'action était appliquée sur le mauvais nœud générant des exceptions.
- Ajout : Mode d'échec rapide en mode action multiples. Si une action échoue ou renvoi un code de retour, plus aucune action n'est traitée.
- Correction : Lorsque "Utiliser dans un nouveau Job a distance" était invoqué pour une liste d'ordinateurs cibles favorite et qu'un nouveau Job à distance était sélectionné, la liste des machines cibles n'était pas mis à jour.
- Correction : Lorsque l'emplacement des favoris était modifié RemoteExec sauvegardait les Job à distance, les actions à distance et les listes de machines cibles toujours dans l'emplacement par défaut.
- Correction : Si des fichiers non XML se trouvaient dans les dossiers des favoris une exception était générée. Désormais les fichiers non XML sont juste ignorés.
- Correction : Le mot de passe du compte exécutant RemoteExec n'était pas sauvegardé/chargé pour le contexte d'exécution "Administratif avancé" même si "Enregistrer le mot de passe" était sélectionné.

5- Quoi de neuf dans RemoteExec 5 beta 2 (Novembre 2010)
--------------------------------------------------------
- Ajout : Le mode ligne de commande a été réintroduit. La commande RemoteExec est désormais un programme console permettant de suivre l'exécution plus facilement.
- Ajout : Le mode ligne de commande permet de définir complètement une exécution de fichier à distance, une action système (Arrêt, redémarrage, fermeture de session, verrouillage de session, réveil) et un popup sans utiliser une action à distance sauvegardée.
- Amélioration : Le chemin du répertoire de RemoteExec a été rajouté dans les chemins de recherche permettant d'invoquer la commande RemoteExec de n'importe quel endroit sans devoir spécifier le chemin complet.
- Amélioration : Les informations de réveil (adresse MAC, sous réseau) sont automatiquement scannées pour toutes les machines ou une exécution à distance a lieu.
- Ajout : Possibilité de changer l'emplacement des favoris (ex: pour les partager sur un serveur).
- Ajout : Possibilité d'ajouter une liste de machine cible directement depuis "Mes machines cibles".
- Correction : L'identificateur de Job n'était pas sauvegardé lors de la planification d'un job à partir des résultats d'une exécution.
- Mise à jour : De l'aide en anglais et d'un nouveau guide de démarrage rapide en anglais et en français
- Ajout : Possibilité d'importer les fichiers actions de RemoteExec 4.
- Ajout : Possibilité d'enregistrer des actions à distance dans le système de fichiers.
- Ajout : L'operation "Rapatrie les infos de réveil" dans l'action système pour récupérer les adresses MAC et les sous réseaux des ordinateurs cible

6- Quoi de neuf dans RemoteExec 5 beta (Novembre 2010)
------------------------------------------------------
- Ajout : Nouvelle interface à onglets. Les nouvelles exécutions à distance sont lancées dans un nouvel onglet.
- Ajout : Insertion des événements dans une base de données afin de conserver un historique exhaustif des exécutions
- Ajout : Possibilité de sauvegarder les configurations d’actions à distance dans un répertoire de favoris afin de les réutiliser rapidement sur des systèmes cibles différents
- Ajout : Possibilité de sauvegarder les configurations de « jobs » à distance afin de les ré-exécuter facilement (un job = une action à distance + la liste des systèmes cibles)
- Ajout : Possibilité de relancer un “job” à distance seulement sur les systèmes indisponibles lors des précédentes tentatives d’exécution
- Ajout : Possibilité de sauvegarder des listes prédéfinies de systèmes cibles afin de les réutiliser rapidement avec des actions distantes nouvelles ou sauvegardées
- Ajout : Possibilité d’afficher un graphique montrant la proportion de succès et d’échecs d’une exécution à distance
- Ajout : Possibilité d’afficher un rapport détaillant le statut d’une exécution à distance sur tous les systèmes cibles d’un « job » 
- Ajout : Possibilité de planification d’un “job” pour en automatiser l’exécution 
- Ajout : Possibilité de rappeler facilement les actions et “jobs” récents. 
- Amélioration : Le nouveau moteur d’exécution de RemoteExec n’utilise plus RPC et DCOM. Seuls ICMP (ping) et le partage de fichiers et imprimantes Micrsoft (SMB TCP 445) sont requis.
- Ajout : Toutes les communications sont cryptées à l’exception des transferts de fichiers, l’accès à distance aux bases de registre et redirection de console en entrée et sortie.

21- Quoi de neuf dans RemoteExec 3.02
------------------------------------
- Ajout: Le service pack 1 pour Windows XP peut désormais être déployé avec RemoteExec mais il faut utiliser la version SP1a disponible à l'URL suivante :
http://www.microsoft.com/downloads/details.aspx?FamilyID=83e4e879-fa3a-48bf-ade5-023443e29d78&DisplayLang=en
Rappel: La première version du SP1 ne peut pas être déployée avec RemoteExec car dans certains cas l'installation bloque ce qui peut provoquer la corruption de la machine distante après un redémarrage (écran bleu).
- Correction : Le bug du browser d'ordinateurs mentionné dans la version 3.01 n'était pas réellement corrigé. C'est maintenant le cas.
- Correction : Autre bug dans le browser d'ordinateurs. Dans certains cas les machines étaient listées sous le mauvais domaine.
- Correction : Bug en mode console. Symptôme: Echec de l'exécution 0006 le handle est invalide.
- Ajout: RemoteExec ping chaque computer avant de lancer l'exécution pour éviter de long délais avec les machines indisponibles.
Si nécessaire cette fonctionnalité peut-être désactivé avec la valeur du registre suivante:
HKEY_LOCAL_MACHINE\SOFTWARE\ISDecisions\RemoteExec\NoPing = 1 (DWORD)
- Amélioration: Lors de la planification d'une exécution, si le mot de passe de l'utilisateur a déjà été rentré RemoteExec configure automatiquement le compte d'exécution de la tâche.
- Amélioration: Après la configuration d'une exécution planifiée RemoteExec demande (et sauvegarde) si nécessaire (execution en mode administratif avancé) le mot de passe du compte d'exécution de la tâche. Cela évite que l'exécution bloque car RemoteExec demande le pot de passe.

24- What's new in RemoteExec 3.0 beta 4 (October 24 2002)
--------------------------------------
- Added: An interface to easily schedule RemoteExec tasks (Context menu > View > Scheduler). Not yet documented in the help file.
- Fixed: In previous versions actions that ran correctly with the interface don't automatically ran in command line mode.
- Ability to deploy msi package on Windows NT 4 computers. Not yet updated in the help file.
- Added: In "full administrative" mode the profile of the user (HKEY_CURRENT_USER) is now loaded before starting the execution to improve the execution success for some programs.
- No fixed expiration date: This release expire 15 days after the installation like in the final release.

27- What's new in RemoteExec 3.0	beta	(September 03 2002)
------------------------------------
- Three new specialized actions:
Service Pack deployment:
	- Parse the file name in order to retrieve (if posssible) the SP number and the OS version.
	- Install only on concerned computers
Hotfix & Updates deployment
	- Find automatically in most cases (for microsoft updates) the arguments in order to run the installation silently
	- Install the hotfixes only if needed
	- Parse the file name in order to retrieve (if possible)  the SP number, the OS version and the hotfix name.
Windows Installer package deployment
	- Ability to install/uninstall/repair all Windows Installer packaged applications.
	- Ability to use transform files
	- Ability to generate log files
- New execution mode in addition of the already existing interactive and administrative(non interactive) modes: The "full administrative" mode allows to run the process on the remote computer without downgraded rigths. So the remote process can acces to the network shares.
- Ability to retrieve locally the command line console for command line tools and scripts (like a telnet).
- A mini scanner in order execute the action only on selected computers according the OS version, the OS level (workstation/server), the service pack number, the Internet Explorer version, and a value in the registry
- Ability to reboot only computers that need a reboot
